# agent-tokens

Extract and chart token usage across AI coding agents. Currently supports
[Claude Code](https://docs.anthropic.com/en/docs/claude-code),
[OpenCode](https://opencode.ai), and
[omp](https://github.com/anthropics/omp) (oh-my-pi).

Agents that aren't installed are silently skipped. If you only use Claude
Code, you'll get Claude Code data and nothing else.

## Quick start

macOS / Linux:

```bash
./run.sh
```

Windows (PowerShell):

```powershell
.\run.ps1
```

On the first run this creates a `.venv/`, installs matplotlib and pandas,
extracts token data from every supported agent it finds on the machine, and
writes charts and a summary to `output/`.

Run it again any time to refresh the numbers.

If you'd rather manage your own environment:

```bash
pip install matplotlib pandas
python3 extract.py -o output/usage.csv
python3 chart.py -i output/usage.csv -o output/
```

## What you get

```
output/
  usage.csv                          # one row per LLM step
  summary.txt                        # totals, per-agent breakdown, monthly table
  charts/
    monthly_total.png                # total tokens by month (stacked cache breakdown)
    weekly_total.png                 # same, by ISO week
    monthly_by_agent.png             # grouped bars, one per agent (2+ agents only)
    weekly_by_agent.png
    monthly_cache_breakdown.png      # cache read / write / input / output
    weekly_cache_breakdown.png
    monthly_by_agent_stacked.png     # per-agent subplots (2+ agents only)
    weekly_by_agent_stacked.png
```

Per-agent comparison charts are only generated when data from multiple agents
exists.

## How it works

`extract.py` reads local session data from each agent:

| Agent | Source | Format |
|-------|--------|--------|
| Claude Code | `~/.claude/projects/**/*.jsonl` | JSONL, `assistant` messages with `message.usage` |
| OpenCode | `~/.local/share/opencode/opencode.db` | SQLite, `part` table `step-finish` records |
| omp | `~/.omp/agent/sessions/**/*.jsonl` | JSONL, `message` records with `message.usage` |

The OpenCode DB path respects `XDG_DATA_HOME` and checks the macOS
`~/Library/Application Support/` fallback.

`chart.py` reads the CSV and generates the charts and summary.

## Token columns

Every row in the CSV has:

| Column | Meaning |
|--------|---------|
| `input_tokens` | Non-cached input tokens sent to the model |
| `output_tokens` | Tokens generated by the model |
| `cache_read` | Input tokens served from prompt cache |
| `cache_write` | Input tokens written to prompt cache |
| `total_tokens` | Sum of all four |

## Requirements

- Python 3.10+
- `matplotlib` and `pandas` (installed automatically by `run.sh` / `run.ps1`)
- `sqlite3` Python module (included in standard CPython; the script warns if
  it's missing and you have OpenCode data)

## Adding a new agent

Add a `parse_<name>` function to `extract.py` that writes rows to the CSV
writer, and register it in the `PARSERS` list. The function should return 0
if the agent's data directory doesn't exist. `chart.py` picks up new agents
automatically.
