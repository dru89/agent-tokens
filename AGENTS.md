# AGENTS.md

## What this repo does

Extracts per-request token usage from local AI coding agent session data,
writes it to a CSV, and generates charts and a text summary. Supports
Claude Code, OpenCode, and omp out of the box. Agents that aren't installed
are silently skipped.

## Repo structure

```
extract.py    — finds local session data for each agent, writes unified CSV
chart.py      — reads the CSV, generates PNG charts and summary.txt
run.sh        — bash wrapper: creates venv, runs extract + chart (macOS/Linux)
run.ps1       — PowerShell wrapper: same thing (Windows)
output/       — generated files (gitignored)
```

There are no dependencies beyond Python 3.10+, matplotlib, and pandas. The
runner scripts create a `.venv/` and install deps automatically.

## Key files

- **`extract.py`** — the main file you'd modify to add a new agent. Each
  agent has a `parse_<name>(writer) -> int` function. The parser contract
  (required CSV fields, error handling rules) is documented in the comment
  block above the `PARSERS` list near the bottom of the file. Adding a new
  agent means writing one function and appending a tuple to `PARSERS`.

- **`chart.py`** — reads the CSV and generates charts. It doesn't know which
  agents exist; it works with whatever `agent` values appear in the data.
  Per-agent comparison charts are skipped when only one agent is present.
  You shouldn't need to touch this file to add a new agent.

- **`run.sh` / `run.ps1`** — thin wrappers that create a venv, run
  `extract.py`, then `chart.py`. No logic worth noting.

## CSV schema

Every row in `output/usage.csv` represents one LLM step (one model request/
response):

| Column | Type | Description |
|--------|------|-------------|
| `agent` | string | Agent name (e.g. `claude-code`, `opencode`, `omp`) |
| `timestamp` | ISO 8601 | When the request occurred (UTC preferred) |
| `input_tokens` | int | Non-cached input tokens |
| `output_tokens` | int | Tokens generated by the model |
| `cache_read` | int | Input tokens served from prompt cache |
| `cache_write` | int | Input tokens written to prompt cache |
| `total_tokens` | int | Sum of the four token fields |

Agents that don't distinguish cache read from cache write should set both
to 0 and put the full input count in `input_tokens`.

## Where agents store their data

| Agent | Location | Format |
|-------|----------|--------|
| Claude Code | `~/.claude/projects/**/*.jsonl` | JSONL; `type: "assistant"` lines have `message.usage` with `input_tokens`, `output_tokens`, `cache_read_input_tokens`, `cache_creation_input_tokens` |
| OpenCode | `~/.local/share/opencode/opencode.db` (also checks `XDG_DATA_HOME` and `~/Library/Application Support/`) | SQLite; `part` table rows where `data` contains `"type":"step-finish"` have a `tokens` object with `input`, `output`, `cache.read`, `cache.write`; joined to `message` for `time_created` |
| omp | `~/.omp/agent/sessions/**/*.jsonl` | JSONL; `type: "message"` lines have `message.usage` with `input`, `output`, `cacheRead`, `cacheWrite`, `totalTokens` |

## Adding a new agent

Write a `parse_<name>` function in `extract.py` and register it in the
`PARSERS` list. The function should:

1. Return 0 silently if the agent's data directory doesn't exist.
2. Print warnings to stderr if data exists but can't be read.
3. Wrap all file/DB access in try/except.
4. Write rows using `writer.writerow()` with the fields listed above.

Look at the existing parsers for reference. `parse_claude_code` and
`parse_omp` show the JSONL pattern; `parse_opencode` shows the SQLite
pattern with schema validation and read-only connection.

## Running

```bash
./run.sh          # or .\run.ps1 on Windows
```

Or manually:

```bash
python3 extract.py -o output/usage.csv
python3 chart.py -i output/usage.csv -o output/
```
